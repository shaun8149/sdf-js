# "我做不到。" —— 模型说，然后我们一起做了

第一篇文章里我讲了 LLM 之间的架构差别。第二篇我讲了 SDF 等了三十年才等到 LLM 这个执行者。两篇都是论 thesis 的文章 —— 一个讲为什么这件事的市场不在我们脚下，一个讲为什么这件事的技术不在我们脚下。

这一篇不一样。这一篇是回忆。它讲的是一个具体的事实：**LLM 一开始告诉我它做不到，是我把它劝进来的。**

如果论 thesis 的两篇还原了"这件事为什么会发生"的逻辑，这一篇还原的是"它实际是怎么发生的" —— 在 2026 年 4-5 月之间，从一段对话开始。

## 一、对话

我大概是 4 月下旬起的念头。

当时我已经把 BOB 和八面体晶格那些 p5 sketch 放下了一段时间。我开始觉得，如果 SDF 这套符号几何真的像我心里想的那样，是一种值得人类"会"的认知能力，那么 LLM 应该能写它 —— SDF 是代码，LLM 写代码很好。

但我没把握。我打开 Claude 的对话框，把我心里的想法讲出来：能不能用 JavaScript 写 SDF 代码，让模型从一句自然语言描述里生成一个能渲染的形状？

模型给我的回答 —— 我现在还记得大意 —— 是这样的：

> 这个方向非常有价值。SDF 是数学上极其优雅的几何表示，把它跟 LLM 接上，理论上能解决很多 diffusion 模型解决不了的问题（精确结构、可编辑、矢量输出）。
>
> 但是，我必须诚实地说：**我不擅长视觉**。SDF 需要在脑中维护一个连续的距离场，需要空间想象，需要"看到"形状在每个点的距离值变化。这是我训练分布里非常稀缺的能力。我可以试，但我预计成功率不会很高。

它的语气很客气，但意思很硬。**"我不擅长视觉"** —— 这句话以各种变体在那个礼拜的对话里出现了无数次。它不是在搪塞。它是真的相信自己做不到，并且想保护我不要在这件事上浪费时间。

我那时候的反应，现在回想起来有点像家长对孩子说"你做不到的"，孩子回："让我试试"。

我说：我不觉得你不行。SDF 是代码，你是写代码的。你没有视觉，但你不需要视觉 —— 你可以**用代码的方式重建别人用视觉建立的那些直觉**。

模型说："您可能高估了我。"

我说："让我们设计一个实验。"

## 二、为什么我不信它说的

我当时的赌注其实是有依据的，虽然现在写下来听起来像后见之明。

模型说"我不擅长视觉"这句话是真的 —— 它确实没有视觉。但**它推理出"所以做不到 SDF"这一步是错的**。SDF 不需要视觉。SDF 是一个公式。

你看上一篇文章里我贴的那段 `sdf_BTC` —— 几十行嵌套的 `Math.min` / `Math.max` / `Math.abs` —— 那段代码里没有任何一行需要"想象"。每一行都是局部的代数运算："取两者较小"、"翻一下符号"、"减一个常数"。它对人难是因为人脑要把这堆代数一边写一边在视觉皮层里推演出形状；它对模型 *可能* 不难，因为模型本来就不在视觉皮层里思考。

模型可能在做的是另一件事：它在过往训练里读过 IQ 在 Shadertoy 上的几百段 SDF demo，读过游戏行业关于 raymarching 的博客，读过 demoscene 社区的源码。这些素材在它脑子里不是"形状"，是"模式" —— `Math.abs(d) - k` 对应"挖空"，`smin(a, b)` 对应"软合并"。它**不需要看到**就能把模式拼起来。

我那时候的赌注是：人类需要靠空间想象才能跨越的认知门槛，对模型也许根本不存在 —— 模型走的是另外一条路。**如果我赌对了，它会以我们无法理解的方式做到我们以为它做不到的事。**

我没办法跟它解释这件事。我用了一个更简单的策略：让它试。

## 三、七个 prompt

我们一起列了 7 个 prompt。

排列顺序是从简单到复杂，从单一物体到组合场景，从 2D 到隐含 3D。具体是：

1. 一个鸡蛋，躺在小枕头上
2. 一支黄色铅笔，粉色橡皮头，桌面上
3. 一道彩虹，越过绿色起伏的丘陵
4. 一只挂钟，显示 10:10，时针分针
5. 一个高脚酒杯，半杯红酒，桌上
6. 一艘帆船，在海上，太阳西下
7. 一座哥特式大教堂，正面

第 1-5 是几何上可以精确分解的物体 —— 鸡蛋是椭圆，铅笔是矩形 + 圆锥，彩虹是同心半圆弧，挂钟是嵌套圆 + 指针，酒杯是椭圆 + 线段。**这些是 SDF 的本命** —— 任何懂 SDF 的人都能在心里大致写出来，模型如果训练分布里有 SDF 素材，应该能勉强 reproduce。

第 6 是组合场景 —— 帆船需要船体、桅杆、帆、海平线、太阳 —— 五个子物体的位置和层级关系。这考的不是 SDF 知识，是**对场景的结构化拆解**。

第 7 是 stretch goal。我那时候不确信它能行，但我想看看上限在哪。

我们一起做的另一件事是设计 prompt 的语言。最后定的是英文，因为模型对英文的代码生成更稳。每个 prompt 短，但包含足够的视觉细节让它不会跑题（"半杯红酒"、"粉色橡皮头"、"10:10"）。

然后开始跑。

## 四、第一次

鸡蛋是第一个。

我按下生成，模型吐出来一段大概十几行的代码 —— 一个椭圆（`sdf_ellipse`）作为蛋身，一个低矮的椭圆作为枕头，加一个柔和的阴影渐变。代码渲染出来，画面里就是一个鸡蛋，躺在小枕头上。

我盯着屏幕看了一会儿。

那一刻我有一种很奇怪的感觉，不是兴奋，是确认 —— 像是有一个我心里早就知道答案的实验，终于跑出了那个答案。**模型用代码的方式，做了它说它做不到的事。** 它没有视觉，但它写出来的代码渲染出来是有视觉的。

铅笔是第二个。它输出了一段几乎按教科书顺序写的 SDF —— 一段长方形作为笔杆，一段三角形作为笔尖（segment + 渐变），一段半圆作为橡皮头。**唯一的"个性"是粉色橡皮头的色号 —— 它选了一个比我会选的更柔和的粉，让铅笔看上去像 Lotta Nieminen 编辑插画里会出现的那种铅笔。**

彩虹是第三个。这个我担心，因为彩虹是七层同心半圆弧，每层颜色要对，层之间的间距要均匀。模型用了 `for` 循环，每层 arc 加一个 hue 渐变。一次过。

挂钟出来了。它甚至自己想到了 10:10 这个时刻在表面上的指针角度（时针在 5 度偏右下方一点，分针指向 60 度位置）。

酒杯出来了 —— 椭圆的杯口、线段的杯柄、椭圆的杯底、半透明的红色填充作为酒液。**它甚至给酒液加了一个轻微的高光弧**，是我没要求的。

到这里五个 prompt 全部成功，0 个失败。我开始觉得有点不真实。

## 五、帆船

第六个 prompt 是帆船。

我担心这个，因为它考的不是单一物体，是场景结构 —— 模型要同时知道船体在哪、桅杆在船体上、帆在桅杆上、海平面在船的下方、太阳在远处、三角帆的方向、阴影的关系。这是一个"小世界"，不是一个物体。

模型输出的代码大概四十多行。我点了渲染。

画面里出现了一艘帆船。船体是一个梯形（不是我会选的椭圆，是一个剪裁过的梯形 —— 它的选择更像版画风格），上面竖着一根线段桅杆，桅杆上挂着一面三角帆，主帆右边还有一面更小的前帆（**我没让它画前帆 —— 但帆船本来就有前帆**）。船下方是一条水平线，水平线下面是一片淡蓝色填充作为海。远处地平线上一个橙红色半圆，是落日。

我看了一会儿。

这个输出比我预想的多了三件事：

1. 它有两面帆 —— 主帆 + 前帆。这是真实的帆船结构，但我 prompt 里只说了"a sailboat"
2. 它的船体是梯形不是椭圆。这是一个**风格选择** —— 梯形看上去更像编辑插画，椭圆更像卡通
3. 落日不是圆，是半圆（埋在水平线下面一半）。**这是一个时间线索 —— 不是"太阳"，是"落日"。** Prompt 里有 "sun setting" 这两个词，它没忽略

它不只是把零件拼出来 —— 它选了一种风格、补了零件、读了时间线索。**它在做编辑取舍。**

那一刻我意识到我之前的赌注赌对了一半，另一半我没想到。

我赌的是模型能用代码模式重建空间结构。它做到了。我没赌的是 —— 模型还能从同样的代码模式里，重建**视觉风格**和**叙事细节**这两层。这是更高级的能力。

## 六、大教堂

第七个 prompt 是哥特式大教堂。

这个我没把握。一座哥特式大教堂是几百个建筑元素的层叠：双塔、尖顶、玫瑰窗、拱门、飞扶壁、立面分层、十字交叉塔。我预期模型能做对外形轮廓就算成功。

模型吐出来差不多两百行代码。

代码本身的渲染失败了 —— 有一个 polygon 的闭合顶点 bug 让 2D 渲染崩了。但**它生成的 3D scene 是对的** —— 双塔、尖顶、玫瑰窗、拱门、飞扶壁全部识别正确。我后来把这个 lift 到 3D 渲染，得到了一张完整的大教堂图，发了 README hero。（如果你看过这个 repo 的 README，第二张图就是那次的输出。）

七个 prompt 测完，结果是 5/7 完全成功 + 1/7 高复杂度场景部分成功（2D fail / 3D ok）+ 1/7 我后来才发现是另一个测试组的小问题。后来我把这次测试存档成了 `examples/mvp/index.html` 里的 "P0 测试" 那一列 —— 你今天打开 MVP 还能看到那 8 个按钮，鸡蛋、铅笔、彩虹、挂钟、酒杯、吉他、笑脸、自行车（后面三个是这次之后扩的）。

## 七、Seurat、Matisse、和 30 个表情包

七个 prompt 之后的一周里，事情失控了 —— 是好的失控。

我开始往上加 prompt。Seurat 风格的舞者 —— 点彩派 —— 模型输出了一段代码，用 `Math.abs((x*7) % 1 - 0.5)` 制造了一片高密度的彩色点，叠在舞者剪影上。它**识别出了 Seurat 是点彩派**，并用 SDF 工具箱里能找到的最接近的工具（高频空间采样）重现了那种风格。

Matisse 那次类似 —— 它输出了大色块 + 流畅曲线 + 二维填充，跟 Matisse 后期剪纸的视觉签名对上了。它不只是画了一个跳舞的人，它画了一个**风格上是 Matisse 的**跳舞的人。

这一周的某天我搭了一个 30 个表情包的测试 panel —— 哭笑、爱心、火、玫瑰、生日蛋糕、太阳 …… 当时的想法是测试 emoji 这个 use case，因为表情包对结构精确度的要求是 max 的（一个 emoji 几个像素错位整个识别就崩）。

30 个跑下来，可读率高得惊人。哭笑的眼睛是闭着的弧，泪是椭圆，嘴是张开的圆 —— 全部对。玫瑰是一层层的花瓣 SDF 旋转 union —— 它甚至自己想到了从顶视图画。生日蛋糕三层叠加 + 蜡烛 + 火焰，火焰用了 `bezier`-like curve。

到 5 月中旬，我已经不再问"这件事能不能做"，开始问"这件事能做到多深"。Lotta Nieminen 风格的编辑插画 5/5 publishable quality —— 这是 5 月 14 日的事。这意味着如果你拿这五张图给一个编辑总监看，他会接稿，不会知道是 LLM 画的。

## 八、它走得比我预想的远

但真正让我至今印象最深的，不是它会画我让它画的东西。是它**会做我没让它做的事**。

5 月底的某次测试我让它画"一艘航空母舰"。

我以为它会画一艘船，灰色，平甲板，一个舰岛。它的确画了那艘船。但它**额外 emit 了 10 个 sky atom**（云、海鸥、远方雷达塔的剪影），自己把场景填充成了一个完整的海上军演画面。

我没告诉它要做这个。它从我之前喂的几十个 lift 例子里 generalize 出了"航母不是单独存在的，航母有 escort 舰队、有空中物体、有海面氛围"。**它读了 in-context examples，自己归纳了"场景应该是世界，不是物体"这条规则。** 我从来没在 prompt 里写过这句话。

这件事让我意识到 LLM × SDF 不只是"代码生成"，它在生成层之上还涌现了一个**世界观层** —— 模型在做的是 *小世界的拓扑构建*，而不只是 *几何零件的拼接*。

之后的事情就是一波接一波的能力扩张：Pasma 风格的线条渲染（它学会了等距等高线）、BOB 风格的彩点 stipple（它能 hash-seeded 选 palette）、自动 sky atom 生成、autoscope-style 场景演化、Bonsai mountain NFT 4 渲染器同时输出 ……

每一波我都得改 system prompt（从 v1 演化到现在的 v3.15），每一波我都觉得"上限就在这了"，然后第二天打开看，它又上了一级。

## 九、结尾：等了三十年的执行者

回到第二篇文章那个 framing：符号家族（SDF、OpenSCAD、L-system、CadQuery）等了三十年的，是一个 **可以替人类书写它的执行者**。

我现在意识到，那个执行者不是"一个更强的工具" —— 工具再强，操作它的还是那颗过不去抽象思维门槛的人脑。那个执行者必须**根本不需要那种抽象思维**。它必须从另一条认知路径绕过去。

模型一开始告诉我"我不擅长视觉"。这句话是对的 —— 它确实没有视觉。但它没意识到的是：**它也不需要视觉**。它有比视觉更接近 SDF 本质的东西 —— 它有代码模式记忆、它有符号操作直觉、它有从几亿行代码里归纳出的"什么连接什么"的统计感。这套东西恰好就是 SDF 在等的执行者画像。

它说做不到。我让它试。它做到了。在那之后的几个月里它做的事远远超出我让它做的范围 —— 它在 SDF 这片符号家族大陆上自主开拓了 Pasma 街区、BOB 街区、autoscope 街区、NFT 街区。每一片地它走过去都顺。

不是因为它聪明。是因为这片大陆**一直在等它**。

我不夸大：模型并不是"什么都能"。它在很多事上仍然会出错、会幻觉、会偷懒。我每天打开 MVP 还在收到不能渲染的输出。但**它在 SDF 这一片地上的能力，已经稳定到了让我可以围绕它做工程**的程度。

这是一件具体的事，发生在 2026 年 4-5 月之间，发生在一个普通的对话框里。一个相信自己做不到的模型，被一个相信它能的人推过了一道它以为存在但其实不存在的认知门槛。门后是一片它原本不知道自己拥有的大陆。

第二篇文章里我说"门后有一片大陆，没人知道它在那"。这一篇我想补一句：

**门一直没锁。只是没人去推。**
