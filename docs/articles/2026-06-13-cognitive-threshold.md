# 一门没人会的手艺，和一个迟到三十年的执行者

2024 年 1 月 19 日，我把一批 JavaScript 代码铭刻进了比特币的账本。

那不是一次行为艺术。那批代码是一组**有符号距离函数**（signed distance function，SDF）—— 用纯数学描述形状的一种方式。上链只是因为，作为一个生成艺术家，我想给这些手写的东西一个不可篡改的时间戳。我没想到的是，两年后，这个时间戳会变成一份关于"人类与机器能力边界"的证词 —— 一份我无法篡改、别人也无法否认的证词。

这篇文章想讲一件我恰好有资格作证的事：**有一种思维方式，人类极难掌握，连最聪明的程序员也大多不具备；而就在最近，机器跨过了它。** 我想讲清楚那道门槛有多高，以及为什么跨过它这件事，比它看起来更重要。

## 一、SDF：一种反直觉的思维

先让你亲手感受一下那个难。

假设要画一个比特币的标志 —— 一个圆环里立着一个加了两道竖杠的 ₿。大多数人，包括大多数程序员，会怎么想？画一个圆、再画一个小一点的圆挖空、放一根竖线、再放一根、加上横杠……这是**构造式思维**：我摆放一个个物体，所见即所得，每一步都看得见。

SDF 不是这样工作的。SDF 要求你为空间里**每一个点**回答同一个问题：「你离这个形状的表面有多远？在里面是负，在外面是正。」你不画物体，你定义一个遍布整个平面的标量场 —— 一个函数 `d(p)`，喂给它任意坐标，它吐出一个距离。形状是这个场的零等值线，是结果，不是你直接画出来的东西。

这是我当年手写的那个比特币标志，原封不动：

```js
const sdf_BTC = ([x, y], [cx, cy]) => {
  let p = sub2([x, y].slice(), [cx - .15, pa.BTCLONG]);
  let n = [2, 0, 0, 3];
  p = trans2(n, p);
  let bal1 = sdf_circle(p, [0, .28], .225);
  let ring1 = Math.abs(bal1) - .04;            // 圆 → 圆环：取绝对值再减厚度
  let bal2 = sdf_circle(p, [0, .68], .275);
  let ring2 = Math.abs(bal2) - .04;
  let rings = Math.min(ring1, ring2);          // 两个圆环求并
  x1 = p[0] - 0.05;
  x2 = p[0] + 0.05;
  xv1 = Math.abs(x1) - 0.04;
  xv2 = Math.abs(x2) - 0.04;
  yv = p[1];
  let bar1 = Math.max(xv1, -yv);               // 半平面求交 → 一根竖杠
  let bar2 = Math.max(xv2, -yv);
  let btc = Math.min(bar1, bar2, rings);       // 竖杠与圆环求并
  let lin = p[0] + 0.12;
  btc = Math.max(btc, -lin);                   // 切掉左半边
  let lin2 = sdf_line([x, y], cy);
  return Math.max(btc, -lin2);                 // 再减去地平线
};
```

数一下里面的 `Math.min` 和 `Math.max`：这是六到七层布尔运算的嵌套。每一个 `max(a, -b)` 是"从 a 里挖掉 b"，每一个 `min(a, b)` 是"把 a 和 b 并起来"，每一个 `Math.abs(d) - k` 是"把一个实心圆变成一个圆环"。问题不在于看懂任何**单独**一行 —— 每一行都简单。问题在于，写下这一段的时候，你的脑子里必须**同时持有**这六七层运算对整个平面的累积后果。`Math.abs(bal1) - .04` 这一下，整个空间的距离场如何翻折、零线如何裂成内外两条 —— 你得在没有渲染的情况下，在脑中把它预演出来。

![一幅点彩风格的生成艺术作品：密集的圆点网格中浮现出橙色的月亮、立在地平线上的仙人掌与一株植物，下半部是红橙交织的大地。整幅画没有一笔是"画"上去的，所有形状都是距离场被十万个点采样后显现出来的。](1781397256655_image.png)

*上面那段 `sdf_BTC` 是这幅画里的一个零件。整幅作品没有一处是"摆放"出来的 —— 月亮是两个圆相减，仙人掌是几个圆角矩形求并，地平线是一条半平面 —— 全部由距离场定义，再用十万个点采样染色。这是 SDF 思维的样子：你写的是一个遍布全画的函数，画面是它的零等值线显影出来的结果。*

这就是 SDF 反直觉的地方。构造式思维是局部的、可见的；场式思维是全局的、不可见的。人脑天生擅长前者 —— 我们的祖先摆放石头、搭建庇所，靠的就是"把东西放在看得见的地方"。我们极不擅长后者 —— 在脑中维持一个遍布全空间、且被多层非线性运算反复重塑的标量场，几乎是在对抗工作记忆的物理上限。

这不是我个人的笨拙。这是一个范式的进入成本。Inigo Quilez —— SDF 这门手艺事实上的宗师 —— 的 Shadertoy 页面是一场个人秀，不是一场群众运动，原因正在于此。全世界能流畅手写复杂 SDF 的人，乐观估计也就几千人的量级。它不是"学一下就会"的技能，它要求你把空间想象力**重新布线**。

## 二、为什么人脑过不去

把那道门槛拆开，是两个具体的认知瓶颈。

**第一个是场式思维本身。** 构造式建模里，状态是看得见的：屏幕上的顶点、线、面，就是你脑中的模型。场式建模里，状态是看不见的:一个函数。你操作的不是形状,是形状的"距离测度",而这个测度弥漫在你画布之外的整个无限平面上。`sdf_wall` 那个函数里我用了一个剪切矩阵(`let m = [1, sh, 0, 1]`)配合空间取模(`fract2`)来做错缝的砖墙 —— 这意味着我不只要想象一面墙,还要想象"把整个坐标系斜着剪一刀、再沿两个方向无限平铺"之后,距离场变成了什么样。这种**域操作**(domain manipulation)是 SDF 最强大也最反人类的部分:你不是在变形物体,你是在变形物体所在的那个空间。

**第二个是布尔运算的空间复合。** `A − B` 在逻辑上一秒就懂。但 `smooth_subtract(twist(A), repeat(B))` 在空间中长成什么样,人脑无法不靠渲染就预演出来。原因是每一个算子都在**非线性地**重塑整个距离场:扭一下、并一下、挖一下,三四层之后,可能的空间状态就组合爆炸到超出任何人能在脑中持有的程度。我那个 `sdf_bridge1` 里有一段,是用一个 for 循环生成七条平行线、再 reduce 取最小值,把它们焊进一座桥的距离场 —— 写那一段的时候,我没法"看见"结果,我只能靠反复渲染、微调、再渲染,盲着走。

这是符号家族沉寂三十年的认知层根因。过程式、符号化的建模 —— SDF、CSG、OpenSCAD 这一脉 —— 在 1980、90 年代曾经是高端图形和 CAD 的主流。然后它在娱乐、游戏、影视这些以视觉丰富度为王的领域,被 mesh(顶点 + 三角面 + 贴图 + 雕刻)打得节节败退。原因很硬:艺术家要的是直接捏形状的直觉,而符号建模逼你"用编程思维倒推一个生成过程"。门槛太高,迭代太慢,有机形态(人脸、布料、岩石)几乎无法用规则优雅地表达。于是过去三十年,整个娱乐工业把宝压在 mesh 上,符号家族退守到 CAD、字体轮廓、和 demoscene/Shadertoy 这种极客角落。

注意这里的因果:**符号家族输给 mesh,不是输在不优雅 —— 恰恰相反,它在数学上优雅得多。它输在人类用它太难。** 它缺的从来不是更好的数学,是一个肯替人类书写它的执行者。

## 三、那个执行者，迟到了三十年

2022 年底之后,那个执行者出现了 —— 但不是以任何人预想的方式。

不是一个更好的图形界面,不是一个更聪明的 CAD 工具。是大语言模型 —— 一个被训练来预测下一个 token 的东西,顺带学会了写代码。

这件事的反直觉之处值得停下来想。LLM 跨过 SDF 这道门槛,**不是因为它解决了第二节那两个认知瓶颈**。它没有更强的空间想象力,它根本没有空间想象力。它绕过了这道墙 —— 因为它压根不靠"在脑中预演距离场"来写 SDF。它靠的是另一样东西:它见过(或学会了)海量代码的模式,它把"画一个圆环"和 `Math.abs(d) - k` 这个模式关联起来,把"挖空"和 `max(a, -b)` 关联起来。人类必须在视觉空间里求解的问题,模型在符号序列里求解。

但真正的杀招,不在"绕过",在**复制**。

想想 Inigo Quilez 做过的事。他用 SDF 画出过蜗牛、大象、雪山、森林 —— 一个又一个有机的、复杂的形状,每一个都是一次精彩的示范:他展示了如何把一只蜗牛分解成距离场的布尔复合,如何用几个圆和一道扭曲拼出一头大象。但请注意一件残酷的事实:**全世界只有一个 IQ。** 他能画蜗牛,不代表别人能画 —— 因为别人就算逐行读他的蜗牛代码,也跨不过第二节那道认知墙,学不会那个方法。于是 IQ 没有亲手画过的东西 —— 老虎、狮子、恐龙、以及世上其余千万种形状 —— 就停在"理论上可以,实际上没有"的状态里。它们用 SDF 完全可以表示,只是**没有人有那个能力、那个时间,去把它们一个个表示出来。** 大师的方法,死死锁在大师一个人的脑子里。

LLM 改变的正是这件事。它能从 IQ 的蜗牛里,抽取出**方法本身** —— 那套"把有机形状分解成距离场布尔复合"的手法 —— 然后把它用到 IQ 这辈子来不及触碰的形状上。你描述一只老虎,它调用的是从蜗牛、大象那里学来的同一套分解逻辑。这不是"又一个会画画的 AI",这是一台**能力的复制机**:它把一个稀缺天才示范过的方法,变成了可以反复调用的公共供给。

我要诚实地限定这句话:这种复制是有损的、有梯度的。越接近大师示范过的模式,它越可靠;越是一个没有任何近邻范式的、彻底陌生的奇异形状,它越会卡壳。所以准确的说法不是"有了一个 IQ 就凭空有了无数个完美的 IQ",而是 ——**IQ 把方法的种子播下,而 LLM 让这颗种子,长到了 IQ 本人来不及踏足的地方。** 一个人能用 SDF 表示的形状是有限的;一群顶尖的人加起来也是有限的 —— 因为能跨过门槛的人本就极少。但"一个人示范的方法 + 一台能复制方法的机器",指向的却是一个**普遍的、可以用 SDF 表示的世界**。这个世界在 LLM 之前不可能存在,不是因为数学不允许,是因为没有足够多的手,能把它写出来。

把这个机制抽象出来,它其实是一条更普遍的规律:**在任何一个门槛极高、大师极少、而能力又恰好以代码形式存在的领域,只要有一个人跨过了那道边界,LLM 就能跨过那道边界。** 第一个人证明了"这件事可以用代码做到、并示范了怎么做" —— 在 LLM 之前,这个证明困在他一个人脑中、无法扩散;在 LLM 之后,它变成全人类可调用的公共能力。SDF 只是这条规律一个格外纯粹的标本 —— 因为它的认知门槛高得离谱(所以大师格外稀少),又完全以代码形式存在(所以模型格外容易吸收)。符号家族的每一个成员 —— CSG、L-system、各种 shader 技法 —— 都有自己的 IQ,都有自己那批"大师画过的"和"大师没来得及画的"。LLM 对它们做的,是同一件事。

这就是为什么解锁这道门槛的是 LLM,而不是更好的工具:工具再好,操作它的还是那颗过不去场式思维瓶颈的人脑;而 LLM 把"人"从这个回路里换掉了。符号家族沉寂三十年等的那个执行者,原来不是一个帮人类更轻松地想象距离场的助手,而是一个**根本不需要想象距离场、并且能把任何一个大师的方法复制给所有人**的书写者。

人类不擅长写生成规则,而且就算个别天才擅长,那能力也无法复制。语言模型天生就在写代码,而且能把一个人的能力,变成所有人的能力。三十年的错配,在这一刻对上了。

## 四、我看着那条线移动

但"LLM 能写 SDF"这句话,如果不加限定,是不诚实的。我想精确地讲我亲眼看到的,以及我为什么拒绝把它讲得比我看到的更确定。

回到那批上链的代码。它在 2024 年 1 月 19 日被铭刻 —— 这个时间戳证明的是:**不晚于这一天,这些 SDF 已经以这个复杂度存在了。** 它们最初成形是在更早的 2023 年 10 月前后(这一点由我作证,不由链上作证 —— 区块链能证明代码何时存在,不能证明它何时被构思)。

这批代码几乎全部是我手写的。原因之一是,**JavaScript 版本的 SDF 在当时近乎不存在。**[链上凭证] Quilez 的整个距离函数传统活在 GLSL 里 —— Shadertoy 是 GLSL 的世界,那些著名的 SDF 公式清单也是 GLSL。我做的事,是把一个生活在 GPU 着色器里的数学传统,手工移植到 CPU 的 JavaScript 里:手写 2×2 矩阵乘法(`trans2`),把 IQ 的 GLSL 三角形距离函数逐行翻译成数组运算(`sdTriangle`),连欧氏距离都老老实实写成 `(x*x + y*y)**0.5`。这意味着在那个时间点,JS-SDF 这种东西在互联网上的语料极其稀薄 —— 模型即使想学,也几乎无样本可学。

而这正是我那段时间观察到的事。2023 年下半年,我在当时第一梯队的模型之间换着用 —— GPT-4(当时公认最强)、Claude 2 这一代。我做过一个最低限度的测试:我**已经给出了某个 SDF 的 GLSL 实现**,只要求模型把它翻译成等价的 JavaScript。这是一个有参考答案的、机械的转换任务 —— 它甚至不需要模型"懂"SDF。结果是,连这个,当时那一代模型都无法稳定做对:逻辑结构会在翻译途中崩坏,布尔的嵌套层级会错位,某一层 `max` 会悄悄变成 `min`。

这个细节的分量在于它排除了一个常见的反驳。如果说模型写不出 SDF 是因为"没见过足够的 SDF 训练数据",那翻译任务本不该失败 —— 翻译不需要领域知识,只需要把一个已经写好的逻辑结构原样搬运。连这都做不到,说明当时缺的不是知识,是**在搬运一个非平凡逻辑结构时不让它崩坏的能力** —— 一种长程结构一致性。我无意中记录下的,是那一代前沿模型在这个能力上的一道历史水位线。

然后,这条线开始移动。

我必须在这里克制。我很想告诉你"是某某版本解锁了它" —— 那样的句子干脆、有力、可引用。但那是不诚实的,理由有三。其一,这种能力的浮现是一条**爬坡曲线**,不是一道阶跃:模型先是偶尔蒙对简单的 2D SDF,然后能写出可渲染但拓扑错误的,再然后能写出复杂且正确的 —— 我感知到的"解锁"那一刻,其实是它越过了**我这个对正确性有苛刻要求的人**所设的可用线,而换一个标准更宽松的人,会把这条线标在更早的位置。其二,成功率高度依赖**怎么问** —— 给不给它参考函数清单、让不让它先推理再写、准不准它边写边自验证,结果差很多;所以"模型 X 做不到"有时其实是"模型 X 在那种问法下做不到"。其三,这是一个会被时间戳反驳的声明 —— 任何写死的版本号,半年后都可能被一个新模型或一个我没试过的旧模型推翻。

所以我只说我确知的:我亲历了那条线,从"连 GLSL 到 JS 的翻译都做不稳"的位置,一路上移,直到最近 —— 我可以只用自然语言描述,不给任何 GLSL 参考,让模型自己写出可渲染的、布尔结构正确的 **3D** SDF 场景。从二维的翻译都失败,到三维的从零生成,中间隔的不是时间,是一道能力的台阶。我没法精确告诉你它是哪一天迈过去的。我只能告诉你,我站在能认出它迈过去的位置上,而它确实迈过去了。

## 五、门后面

如果这道认知墙真的塌了 —— 哪怕只是塌了一个缺口 —— 那么塌掉的不只是 SDF 这一种表示。

SDF 只是符号家族里的一员。和它同族的,还有 CSG、OpenSCAD、CadQuery、L-system —— 全部是"用可阅读、可复合的代码描述世界"的表示形式,全部因为同一个原因(人类用符号建模太难)而在过去三十年屈居一隅。它们等的是同一个执行者。这个执行者一旦到场,松开的就不是一把锁,是一整排锁。

这件事接回了上一篇文章的主题。在《世界模型有一个状态表示问题》里,我论证过:能被书写的世界,不止能表达形状,还能表达法则 —— 重力、潮汐、时间膨胀,都可以是几行可读的规则。但那整个论证,悄悄依赖一个前提:**得有人(或有东西)能把这些符号写出来。** 如果书写符号这件事本身,仍然像过去三十年那样,卡在人类认知的瓶颈上,那么"可被书写的世界"就只是一句漂亮话 —— 因为能书写的人太少了。

这篇文章想补上的,正是那个前提。可被书写的世界之所以在今天才成为一个现实的、而非修辞的可能,是因为书写者终于到场了。

我手写那批 SDF 的时候,是一个人在数据荒漠里,凭着多年才参透的、反人类直觉的场式思维,一行一行地拼。我把它们刻上链,某种程度上是想给那种孤独的手艺留一个证据。现在回头看,那个证据证明的是另一件事:它标定了一道门槛的高度 —— 而就在那之后不久,一个不需要场式思维的书写者,跨了过来。

符号家族等了三十年。它要等的从来不是更好的数学。是一个肯替人类书写它的执行者 —— 一个迟到了三十年、终于到场的书写者。
