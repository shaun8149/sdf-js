## 解剖 (recipe)

1. **折叠 = 剖 + 镜像**: 顶点按折线侧别分类, 跨侧的边插入交点切出
   子多边形; 然后把一侧沿折线法向**反射** (可附加平移) — "折"是
   剖分 + 2D 镜像的平面拓扑操作, 没有任何 3D。镜像才是"折",
   剖分只是"切"。
2. **着色 = 随机明度 + 方向纹理**: 每 facet 从调色板组随机取色 +
   随机明度偏移, 再叠方向性纹理填充 (leaf/dots/evenLines/perspective,
   沿填充方向 d9 渐变)。色阶被眼睛读成纸的朝向 (与 L19 Screens 的
   facet 亮度同一效果), 但机制**不是**"按折叠深度渐变" — 原作没有
   折叠深度追踪。
3. **折线是预生成批量作用的**: 折线由 d2/d3/d4/d5/foldOver/radial
   等生成器**预先批量生成** (每档 10~25 条), 逐条作用于全部被穿过的
   facet — **不存在"永远剖最大 facet"的贪心规则** (那是我们发明的;
   全脚本唯一按面积排序处只用于把样式对象按面积序配给 facet)。
4. **折轴角度全随机**; "轴对齐 ± jitter" 存在但在**填充线方向 d9**
   上 (`down: π/2±0.1` / `flippy: ±π/2±0.1`) — 一读把 fill 方向的
   idiom 错安到了折轴上。

## Port: paper-folds 家族 (registry.js, DECOR_V=1 下新增)

区域矩形当纸, K 次"剖最大 facet", OKLab 深度渐变 + 折痕描线。
人格 = 剖分次数 × 折轴抖动 × 色调展幅。适配 organic/hr (软拼贴感)。
与 shard-mesh (碎三角) / block-mosaic (网格) 的差异: **少而大** 的 facet。
**归因更正**: 家族实现的"贪心剖最大 / 轴对齐折轴 / 深度渐变着色"
三条规则均**不是 ORI 的机制** (见解剖 2-4), 属笔记一读的发明;
且家族只剖不镜像, 丢了"剖+翻折"的折纸签名。装饰效果自立, 但
provenance 叙述需按此更正 — 家族二修待 DECOR_V 纪律裁决。

## 一句话学到的

"折"不需要 3D — 剖分给形, 镜像给折, 随机明度加方向纹理给朝向。
