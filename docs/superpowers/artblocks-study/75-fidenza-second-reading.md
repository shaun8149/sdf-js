# 第七十五课 (二代研读): Fidenza 的渲染层 — fat ribbon 与分段色块

- 触发: user 目检指出一代 port (flow-ribbons, L2) "stroke 不对, 一段一段的
  颜色没做出来" 并提供全注释源码研读 (仍守 recipe-only: 学结构, 零代码搬运)

## 一代 port 漏掉的四个渲染 idiom

1. **fat ribbon**: 曲线是"脊线→fatTop/fatBot 法向偏移→闭合厚形", 从来不是
   描边折线。粗细来自 7 档**离散**等级 (z0=2 … z7=320 @2000 设计宽) 按
   变体加权抽取 — 混合尺度, 绝不连续取值。
2. **分段色块 (fSeg)**: 厚带内部沿弧长切成 4-32px 的短段, **每段独立**
   从加权色池重新抽色, 画成上下边缘间的闭合四边形 — "一段一段的颜色"
   即 Fidenza 的签名。段长本身也是加权抽取 (pSL)。
3. **加权色池的纪律**: luxe 池里淡色/新闻纸色占大头 (~55%), 饱和色只做
   点睛 — 场面由合唱队抬着, 高音偶尔出来。背景色按变体尺寸调整。
4. **碰撞留隙**: 空间分区碰撞检测, 曲线相遇即止 (含起步前瞻: 不够
   2×宽度就不开画) — 编织感来自互相让路 (与 L2 结论一致, 这次连
   最小长度门也带上)。

## 产物: banded-ribbons 家族 (registry.js)

脊线走场 (低曲率) → 法向 fatten → 弧长分段逐段抽色 → 圆帽 → 发丝描边;
色池 = 主题色 ×1 + OKLab 半淡 tint ×1.6 + 近底色中性 ×2.2 (加权);
粗细档位 [4..110px] 按人格加权; 网格碰撞 + 2.2×宽度最小长度门。
alpha 公式 fill×1.9: subtle 守 0.1 (守卫), artwork 0.96 (封面成画)。

## 一句话学到的

一代 port 学了"构图靠让路", 二代才学到"身体是厚的、颜色是分段的" —
recipe 的深度取决于你看代码的遍数。
