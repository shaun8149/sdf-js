### 🟡 Sprint 4 候选改进点 (不阻塞 ship)

**1. Wedge boundary — 纯抽象数字不 conjure metaphor** (L6-6C + L3-3B)

Atlas P2 metaphor 需要"**具象 anchor + 数字**"共存才触发:
- 含具象 (L2-2A carrier / L2-2D jet / L6-6A carriers): 5/6 P2 wedge fires
- 纯抽象数字 (L6-6C SaaS metrics / L3-3B revenue growth): 0/6 P2, LLM 退到 P1 / P3 fallback

含义:
- **金融科技 / SaaS deck**: Atlas 输出跟 Napkin / antvis 同档 (失去差异化, 产品仍 work)
- **国防 / 工业 / 基建 / 物理实体 deck**: Atlas 真护城河 (vector tools 做不出)

Sprint 4 v3.21 候选: 在 prompt 加 SaaS-specific worked example, force metaphor 尝试:
- "\$100M ARR" → 100 个 \$1M dollar bill 堆成 column
- "92% margin" → 92% filled bar vs 8% loss 比例
- "10x growth" → ladder / steps / tower 高度比例

**2. Hierarchy/network 关系类应强推 P3 vector, 不是 P1 SDF** (L5-5B)

LLM 在 org chart 题目上选了 P1 traditional rounded_box → 上半 org tree 对的但下半 SDF 渲染溢出黑块. v3.20 prompt 没说清"hierarchy/network/list 类必须走 P3 vector layout (Sprint 1 v4 atom 不适合 abstract relations)".

Sprint 4 v3.21 候选: 在 prompt 加 "If content describes pure abstract structural relations (hierarchy, network, list, comparison) without literal physical objects, prefer Priority 3 P5 vector layout over Priority 1 traditional SDF — SDF atoms are tuned for physical scenes."

**3. Pipeline 加 JS parse 验证作为门** (L3-3B variant 0)

LLM 偶尔 emit truncated JS ("Unexpected end of input"). pipeline 现在直接存为 ready, iframe 里 eval 才发现错. 建议 pipeline.js 加 `try { new Function(args.code); } catch (e) { mark as error with liftError }` 作为 ready 门, 这样 picker thumbnail 会标记该 variant 错而不是用户切到才看到红框.
